Updated instructions as of Jun 23, 2023 (*new annotated examples added).

In this task, we give you a highlighted word or phrase in a Passage Text. The highlighted phrase refers to some kind of situation -- for example, a dynamic event (like running or kicking) or a state of being (like happiness or redness). We also give you the Source Text from which this Passage was taken. The Passage text is essentially a summary of some situation from the Source Text. We then ask you to do the following:

  1. identify the "event type" of the highlighted phrase in the Passage Text
    Note: This is already pre-highlighted for you in this task based on an earlier task performed by workers on MTurk. If you still strongly feel that the pre-highlighted event type is not correct, feel free to change it.
  2. identify different roles of event in (i) in the Passage Text
  3. identify the different roles of the same event in Source Text
Through a running example, we explain what we mean by event types and roles of an event and how to annotate them. On each of the four sections below click on "show" to show the running example for each section.

    1. Identifying the Event Type (hide)

    Let us look at the task through a running example. We have highlighted one word that refers to a situation (or event) in the Passage Text as shown below:


    He was hired as a Research Scientist by Microsoft.

    Your first task is to identify what type of event is highlighted in the Passage Text from a list of 5 options, as shown below:

    Choosing
    Firing
    Renting
    Hiring
    Appointing

    Clicking on an event type button displays the definition of that event type along with an example of that event type:

    Choosing
    Firing
    Renting
    Hiring
    Appointing
    Active Event Type:
    Hiring
    Event Definition:
    An Employer hires an Employee, promising the Employee a certain Compensation in exchange for the performance of a job.
    Event Example:
    John was hired to clean up the file system.

    For our running example, Hiring is the correct event type.

Additionally, each role has a fixed definition and a fixed example. The definition and example of the active role is displayed below the list of all roles.

    4. General Rules for Annotating Roles (hide)

    4.1 Use the most informative word(s) or phrase of a role in the text.
    In both Passage Text and Source Text role annotations, we want you to capture the most informative word(s) or phrase of a role in the following order of precedence (from most to least informative):
    1. words denoting names (such as "John Smith", "United States", etc.)
    2. nominal decriptive mentions (such as "the 32-year-old man", "the guy with pink hair", "the tallest building" etc.)
    3. pronoun mentions (such as "he", "she", "they", "it", etc. )
    4.2 Add extra roles if needed.

    Keyboard Shortcuts

    ← Select previous word in text
    i Show/hide instructions
